# ЗАКЛЮЧЕНИЕ

В ходе выполнения настоящей квалификационной работы была разработана кроссплатформенная мобильная система информационного обеспечения высшего учебного заведения, реализующая централизованную публикацию и рассылку новостей и образовательных ресурсов с разграничением прав доступа по ролям пользователей. Все задачи, сформулированные во введении в соответствии с целью исследования, были последовательно решены на протяжении аналитического и проектного этапов работы.

В рамках аналитической части был проведён детальный анализ предметной области — изучена организационная структура высшего учебного заведения, описаны характер и направления информационных потоков между её участниками. Установлено, что существующие каналы распространения институциональной информации — официальные сайты, электронная почта, группы в мессенджерах — характеризуются принципиальными ограничениями: отсутствием механизма проактивного оповещения пользователей, разрозненностью источников, невозможностью гарантировать охват целевой аудитории. Совокупные организационные и экономические издержки неэффективной информационной среды создают объективное основание для разработки специализированного цифрового решения.

Сравнительный анализ существующих программных аналогов — Microsoft Teams для образования, Moodle Mobile, платформы ВКонтакте для образования — показал, что ни одно из рассмотренных решений не обеспечивает оптимального сочетания функций публикации институциональных новостей и управления учебными ресурсами в едином лаконичном мобильном интерфейсе при контролируемых затратах на внедрение и сопровождение. На основании проведённого анализа обоснована стратегия самостоятельной разработки специализированного приложения и произведён выбор технологического стека: React Native с Expo SDK 50 — для мобильного клиента; Node.js с Express, TypeScript и Prisma ORM — для серверной части; PostgreSQL — для хранения данных.

В проектной части разработана информационная модель системы, включающая четыре сущности — пользователей, новостей, образовательных ресурсов и категорий — с определёнными ограничениями целостности и политиками каскадных операций. Спроектирована и реализована схема реляционной базы данных в третьей нормальной форме, обеспечивающая отсутствие аномалий обновления и минимальное дублирование данных. Сформулирован контракт REST API, охватывающий эндпоинты аутентификации, управления новостями, ресурсами и категориями.

В ходе практической реализации создана серверная часть приложения, включающая модули маршрутизации, middleware аутентификации и авторизации, контроллеры и сервисы для каждой функциональной группы. Реализован мобильный клиент, включающий экраны аутентификации, главный экран с агрегированной лентой, экраны списков и детального просмотра новостей и ресурсов, административные формы создания и редактирования контента, экран управления категориями. Реализованы механизм аутентификации на основе JWT с автоматическим обновлением токенов через interceptor Axios, хранение учётных данных в Expo SecureStore, ролевое управление видимостью элементов интерфейса.

Работоспособность разработанной системы подтверждена контрольным примером, в котором последовательно воспроизведены основные сценарии использования: публикация новости администратором, создание образовательного ресурса преподавателем и просмотр контента студентом. Проверка на уровне API подтвердила корректность реализации ролевого разграничения: попытка обращения к привилегированному эндпоинту с токеном пользователя без соответствующих прав возвращает ответ 403 Forbidden. Время отклика серверной части при выполнении стандартных операций чтения не превысило двухсот миллисекунд.

Практическая значимость разработки состоит в возможности непосредственного внедрения созданного приложения в деятельность высшего учебного заведения. Предложенная архитектура не является специфичной для конкретного учреждения и может быть воспроизведена с минимальными адаптационными изменениями в любом российском вузе, располагающем базовой серверной инфраструктурой. Принятое решение об использовании технологий с открытым исходным кодом исключает лицензионные издержки и обеспечивает полную независимость от решений коммерческих вендоров. Модульная организация серверной части позволяет расширять систему новыми функциональными блоками — например, модулем опросов студентов или модулем управления расписанием — без изменения базовой архитектуры.

Дальнейшее развитие системы может предусматривать добавление механизма комментирования публикаций, интеграцию с системой управления расписанием учреждения, внедрение офлайн-режима работы с кэшированием данных на устройстве, а также разработку веб-интерфейса администратора для управления системой с настольного компьютера. Каждое из этих направлений может стать предметом самостоятельного исследования, опирающегося на архитектурный фундамент, заложенный в настоящей работе.
