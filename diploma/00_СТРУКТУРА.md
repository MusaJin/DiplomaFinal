# СТРУКТУРА ДИПЛОМНОЙ РАБОТЫ

## Тема: «Разработка мобильного приложения для рассылки новостей, образовательных ресурсов высшего учебного заведения с использованием технологии React Native»

---

## ВВЕДЕНИЕ

- Актуальность: цифровизация высшего образования, рост мобильного потребления информации, отсутствие единой точки доступа к институциональному контенту в большинстве российских вузов
- Объект исследования: процессы информационного обеспечения участников образовательного процесса высшего учебного заведения
- Предмет исследования: мобильное приложение для распространения новостей и образовательных ресурсов вуза на основе технологии React Native
- Цель: разработка кроссплатформенного мобильного приложения, обеспечивающего централизованную рассылку новостей и образовательных ресурсов с разграничением доступа по ролям
- Задачи (8 штук — см. ниже)
- Методы исследования и инструменты
- Научная новизна и практическая значимость
- Обзор использованных источников

---

## ГЛАВА 1. АНАЛИТИЧЕСКАЯ ЧАСТЬ

### 1.1 Технико-экономическая характеристика предметной области
- Университет как объект автоматизации: организационная структура, подразделения, роли участников
- Характеристика информационных потоков: администрация → преподаватели → студенты
- Существующие каналы распространения информации и их недостатки
- Экономические и организационные издержки неэффективной информационной среды

### 1.2 Экономическая сущность задачи и обоснование применения вычислительной техники
- Описание задачи рассылки новостей и публикации образовательных ресурсов
- Входные данные: контент (новости, материалы), метаданные, пользователи
- Выходные данные и ролевая модель: ADMIN, TEACHER, STUDENT — информационные потребности
- Экономический и организационный эффект от автоматизации
- Ограничения ручного распространения информации; обоснование выбора мобильной платформы
- Преимущества мобильного решения перед веб-порталом; статистика использования
- Обоснование разработки собственного решения vs. готовые платформы
- Обоснование выбора PostgreSQL/Prisma и React Native/Node.js

### 1.3 Постановка задачи
- Цель и назначение автоматизированной системы
- Функциональные требования (управление пользователями, новостями, ресурсами, категориями)
- Нефункциональные требования (производительность, безопасность, кроссплатформенность, масштабируемость)
- Общая характеристика организации решения задачи

### 1.4 Анализ существующих разработок и обоснование стратегии
- Обзор аналогов: Microsoft Teams Education, Moodle Mobile, VK Education, Яндекс.Учебник
- Сравнительный анализ по критериям
- Обоснование разработки собственного решения
- Выбор и обоснование технологии проектирования (React Native + Expo)

## ГЛАВА 2. РАЗРАБОТКА И ПРОЕКТИРОВАНИЕ СТРУКТУРЫ МОБИЛЬНОГО ПРИЛОЖЕНИЯ «УНИВЕСТ»

### 2.1 Стек технологий и обоснование проектных решений
- Техническое обеспечение: мобильные устройства Android/iOS, серверная инфраструктура
- Информационное обеспечение: реляционная БД PostgreSQL, структура данных
- Программное обеспечение: React Native, Expo SDK, Node.js/Express/TypeScript, Prisma ORM
- Технологическое обеспечение: клиент-серверная архитектура, REST API, JWT-аутентификация

### 2.2 Информационное обеспечение задачи
- Информационная модель системы и её описание
- Классификаторы и системы кодирования (роли, типы ресурсов, категории)
- Характеристика входных данных и первичных документов
- Характеристика базы данных: таблицы users, news, resources, categories, notifications, device_tokens — поля, типы, связи
- Характеристика результатной информации

### 2.3 Программное обеспечение задачи
- Общие положения: дерево функций, сценарий диалога
- Архитектура системы: схема взаимодействия клиента и сервера
- Серверная часть: структура модулей, маршруты REST API, middleware авторизации
- Клиентская часть: навигация (Stack + BottomTabs), экраны, Zustand store, сервисы
- Описание ключевых программных модулей

### 2.4 Технологическое обеспечение задачи
- Схема технологического процесса сбора, передачи и обработки информации
- Аутентификация: JWT, SecureStore, перехватчики Axios
- Push-уведомления: Expo Notifications, регистрация токенов, отправка
- Организация ролевого доступа

### 2.5 Описание контрольного примера реализации
- Сценарий 1: администратор создаёт и публикует новость, рассылает уведомление
- Сценарий 2: студент получает уведомление, читает новость, переходит к ресурсу
- Сценарий 3: преподаватель просматривает образовательные материалы
- Результаты выполнения контрольного примера
