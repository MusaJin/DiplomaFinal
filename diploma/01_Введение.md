# ВВЕДЕНИЕ

Цифровая трансформация высшего образования, набирающая темп на протяжении последнего десятилетия, затронула практически все аспекты деятельности университетов — от организации учебного процесса до административного управления. Одним из наиболее острых и до сих пор не в полной мере решённых вопросов остаётся проблема оперативного и структурированного информирования участников образовательного процесса. Студенты, преподаватели и сотрудники администрации ежедневно нуждаются в актуальных сведениях об изменениях в расписании, новостях учреждения, официальных объявлениях и учебно-методических материалах. При этом большинство российских вузов по-прежнему использует разрозненные каналы передачи информации — официальные сайты, электронную почту, группы в социальных сетях и мессенджеры, — что неизбежно приводит к дублированию сообщений, задержкам в доставке и невозможности гарантировать, что каждый пользователь получит нужные сведения своевременно.

**Актуальность** настоящей квалификационной работы определяется устойчивым сдвигом в паттернах потребления цифровой информации: согласно данным ряда отечественных и зарубежных исследований в области образовательных технологий, более восьмидесяти процентов студентов обращаются к учебному контенту и институциональным новостям преимущественно через мобильные устройства. Смартфон стал основным персональным коммуникационным инструментом, и игнорирование этого факта при проектировании информационных систем вуза означает принципиальное расхождение между предлагаемым способом доставки информации и реальным поведением аудитории. Появление и широкое распространение кроссплатформенных фреймворков мобильной разработки — в частности, React Native — сделало возможным создание единого приложения, одновременно работающего на устройствах под управлением Android и iOS, что принципиально меняет экономику разработки для образовательных организаций, как правило ограниченных в ИТ-бюджете.

**Объект исследования** — процессы информационного обеспечения участников образовательного процесса высшего учебного заведения, охватывающие сбор, систематизацию, хранение и распространение актуальных институциональных сведений среди студентов, преподавателей и административного персонала.

**Предмет исследования** — мобильное приложение для рассылки новостей и образовательных ресурсов высшего учебного заведения, построенное с использованием технологии React Native и обеспечивающее ролевое разграничение доступа к функциям системы.

**Цель квалификационной работы** — разработка кроссплатформенного мобильного приложения, реализующего централизованную публикацию и рассылку новостей и образовательных ресурсов университета с разграничением прав доступа по ролям пользователей.

Для достижения поставленной цели в работе решались следующие задачи:

1. Провести анализ предметной области: изучить организационную структуру типового высшего учебного заведения, характер информационных потоков между его подразделениями и существующие инструменты информирования участников образовательного процесса.
2. Выполнить сравнительный анализ существующих программных решений аналогичного назначения с целью выявления их ограничений и обоснования целесообразности самостоятельной разработки.
3. Сформулировать функциональные и нефункциональные требования к проектируемой системе, обосновать выбор технологического стека и принятых архитектурных решений.
4. Разработать информационную модель системы, спроектировать схему реляционной базы данных, описать структуру REST API и логику ролевого разграничения доступа.
5. Реализовать серверную часть приложения на основе Node.js, Express и TypeScript с применением ORM-инструмента Prisma и СУБД PostgreSQL.
6. Реализовать мобильный клиент на React Native с использованием платформы Expo.
7. Реализовать модули публикации новостей и управления образовательными ресурсами, систему аутентификации на основе JWT-токенов.
8. Провести тестирование разработанной системы и описать контрольный пример её эксплуатации.

**Методы исследования:** системный анализ — при изучении организационной структуры университета и существующих информационных процессов; сравнительный анализ — в основе обзора программных аналогов; метод объектно-ориентированного проектирования — при разработке архитектуры системы и схемы базы данных; метод прототипирования — на этапе проектирования пользовательского интерфейса. Инструментами реализации послужили React Native и Expo SDK — для мобильного клиента, Node.js с фреймворком Express и языком TypeScript — для серверной части, PostgreSQL и Prisma ORM — для организации хранения данных, библиотека Zustand — для управления состоянием клиентского приложения.

**Теоретическая значимость** работы состоит в систематизации и применении существующих подходов к проектированию клиент-серверных информационных систем применительно к задаче автоматизации информационного обеспечения высшего учебного заведения. Разработанная архитектура демонстрирует практически применимую модель интеграции кроссплатформенной мобильной разработки, реляционной базы данных и REST API в рамках единой системы с ролевым разграничением доступа.

**Практическая значимость** разработки определяется возможностью непосредственного внедрения созданного приложения в деятельность высшего учебного заведения. Система позволяет администрации формировать и публиковать новости, преподавателям — размещать структурированные образовательные материалы различных типов, студентам — получать актуальный институциональный контент в удобном мобильном интерфейсе без необходимости обращаться к нескольким разрозненным источникам. Принятые архитектурные и технологические решения могут быть воспроизведены при создании аналогичных систем в других образовательных организациях.

Теоретическую и методологическую основу работы составили труды отечественных и зарубежных специалистов в области проектирования информационных систем, разработки программного обеспечения и мобильных технологий. Вопросы системного проектирования рассматривались с опорой на работы Г. Буча, посвящённые объектно-ориентированному анализу, и И. Соммервилла в части инженерии программных систем. Принципы проектирования реляционных баз данных изучались на основе классических работ по теории нормализации и практике применения SQL. Специфика кроссплатформенной мобильной разработки исследовалась по актуальным изданиям, посвящённым React Native и экосистеме JavaScript. Степень изученности темы в отечественной литературе характеризуется достаточной проработанностью общих вопросов автоматизации образовательных процессов при относительно слабой представленности работ, непосредственно посвящённых кроссплатформенным мобильным решениям для информационного обеспечения российских вузов, что придаёт настоящему исследованию самостоятельную научно-практическую ценность.
