Ты senior full-stack разработчик и системный архитектор. Твоя задача — спроектировать и реализовать полноценный проект по теме:

**«Разработка мобильного приложения для рассылки новостей и образовательных ресурсов высшего учебного заведения с использованием технологии React Native»**

Нужно сделать **реально запускаемый проект**, а не просто пример интерфейса. Проект должен выглядеть как полноценная дипломная/учебная разработка, которую можно показать, запустить, объяснить по экранам и логике.

## Общие требования

Нужно создать систему, состоящую из:

1. **Мобильного приложения на React Native**
2. **Backend API**
3. **Базы данных**
4. **Простой административной логики для управления контентом**
5. **Поддержки push-уведомлений**
6. **Тестовых данных для демонстрации**

Проект должен быть собран так, чтобы его можно было:

* запустить локально;
* показать как рабочий продукт;
* объяснить по архитектуре;
* использовать как основу для диплома или курсового проекта.

## Основная идея проекта

Приложение предназначено для студентов и преподавателей вуза. Через него пользователь должен:

* получать новости учебного заведения;
* просматривать образовательные ресурсы;
* получать push-уведомления о важных публикациях;
* открывать материалы по категориям;
* пользоваться удобным мобильным интерфейсом.

Администратор должен иметь возможность:

* добавлять новости;
* редактировать новости;
* удалять новости;
* добавлять образовательные материалы;
* управлять категориями;
* инициировать уведомления.

## Важное правило реализации

Не делай проект как набор моковых экранов без логики. Нужен **полноценный рабочий MVP**, в котором есть:

* навигация;
* авторизация;
* получение данных с backend;
* хранение данных в БД;
* админская логика;
* нормальная структура кода;
* понятный README;
* инструкции по запуску.

## Технологический стек

Используй следующий стек:

### Mobile

* React Native
* TypeScript
* React Navigation
* Zustand или Redux Toolkit для состояния
* Axios для API
* React Hook Form при необходимости
* AsyncStorage для локального хранения токена/данных
* библиотеку для push-уведомлений, подходящую для React Native

### Backend

Выбери наиболее адекватный и современный стек, но желательно:

* Node.js
* Express или NestJS
* TypeScript
* Prisma ORM
* PostgreSQL
* JWT авторизация
* Swagger или хотя бы понятная структура API

Если считаешь, что для учебного проекта лучше Express + Prisma — используй его, но обоснуй архитектуру в README.

### Admin

Не делай сложную отдельную админ-панель, если это слишком раздует проект. Можно реализовать один из двух вариантов:

* либо отдельный простой web-admin на React;
* либо набор backend endpoints + seed/admin user, через которые контентом можно управлять;
* либо отдельный раздел внутри приложения, доступный только администратору.

Но обязательно выбери **один реальный и рабочий** вариант и реализуй его до конца.

## Пользовательские роли

Минимум должны быть следующие роли:

### 1. Студент

Может:

* войти в систему;
* просматривать новости;
* читать полные новости;
* просматривать образовательные ресурсы;
* фильтровать материалы по категориям;
* получать уведомления;
* добавлять новости или ресурсы в избранное, если это не сильно усложняет проект.

### 2. Преподаватель

Может:

* делать всё то же, что студент;
* при желании иметь отдельный набор ресурсов или категорий, если это логично;
* можно оставить функционально близким к студенту, но роль должна существовать в системе.

### 3. Администратор

Может:

* создавать, редактировать и удалять новости;
* создавать, редактировать и удалять образовательные ресурсы;
* управлять категориями;
* отправлять уведомления пользователям;
* видеть список опубликованного контента.

## Функциональные модули

Нужно реализовать следующие модули.

* блок с образовательными ресурсами;

### 3. Новости

Сделай:

* экран списка новостей;
* карточки новостей;

### 4. Образовательные ресурсы

Сделай:

* экран списка ресурсов;
* группировку или фильтрацию по категориям;
* открытие материала;

Нужно:

* при публикации важной новости должна быть возможность отправить уведомление;
* если полноценные пуши сложно поднять локально, сделай архитектуру и рабочую заглушку, но реализуй это максимально близко к реальности.

### 6. Администрирование контента

Нужна рабочая логика:

* список новостей администратора;
* создание новости;
* редактирование;
* удаление;
* публикация;
* аналогично для ресурсов;
* управление категориями.

### 7. Профиль пользователя

Сделай экран профиля, где можно увидеть:

* ФИО;
* email;
* роль;
* кнопку выхода.
