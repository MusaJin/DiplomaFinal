# УниВест — Мобильный портал университета

Дипломный проект: **«Разработка мобильного приложения для рассылки новостей и образовательных ресурсов высшего учебного заведения с использованием технологии React Native»**

---

## Описание проекта

**УниВест** — мобильное приложение для студентов и преподавателей вуза, предоставляющее централизованный доступ к новостям учебного заведения, образовательным ресурсам и системе push-уведомлений.

Система включает:
- Мобильное приложение (React Native + Expo)
- Backend API (Node.js + Express + TypeScript)
- База данных (PostgreSQL + Prisma ORM)
- Административный интерфейс (встроен в мобильное приложение)

## Установка и запуск

### Требования

- Node.js 20+
- PostgreSQL 15+
- Expo CLI (`npm install -g expo-cli`)
- Приложение Expo Go на телефоне (для тестирования)

---

### Шаг 1: Клонирование и установка зависимостей

```bash
# Клонируйте проект
git clone <url>
cd uni-app

# Установите зависимости backend
cd backend
npm install

# Установите зависимости mobile
cd ../mobile
npm install
```

---

### Шаг 2: Настройка базы данных

Создайте базу данных PostgreSQL:

```sql
CREATE DATABASE uni_app;
```

Создайте файл `.env` в папке `backend`:

```env
DATABASE_URL="postgresql://postgres:YOUR_PASSWORD@localhost:5432/uni_app"
JWT_SECRET="your-secret-key-minimum-32-chars"
JWT_EXPIRES_IN="7d"
PORT=3000
```

---

### Шаг 3: Миграция и заполнение базы данных

```bash
cd backend

# Создать и применить миграции
npx prisma migrate dev --name init

# Сгенерировать Prisma Client
npx prisma generate

# Заполнить тестовыми данными
npm run prisma:seed
```

---

### Шаг 4: Запуск backend

```bash
cd backend
npm run dev
```

Сервер запустится на `http://localhost:3000`

Проверка: `http://localhost:3000/health`

---

### Шаг 5: Настройка мобильного приложения

Откройте файл `mobile/src/services/api.ts` и укажите IP-адрес вашего компьютера:

```ts
// Для Android-эмулятора:
const API_URL = 'http://10.0.2.2:3000';

// Для физического устройства (Expo Go):
const API_URL = 'http://192.168.1.XXX:3000';  // ваш локальный IP

// Узнать свой IP:
// Windows: ipconfig | findstr IPv4
// Mac/Linux: ifconfig | grep inet
```

---

### Шаг 6: Запуск мобильного приложения

```bash
cd mobile
npm start
```

Откроется QR-код. Отсканируйте его приложением **Expo Go** на телефоне.

## API документация

### Аутентификация

```
POST /auth/login
Body: { email, password }
Response: { token, user }

GET /auth/me
Headers: Authorization: Bearer <token>
```

### Новости

```
GET    /news                        # Список новостей
GET    /news/:id                    # Новость по ID
POST   /news                        # Создать (ADMIN)
PATCH  /news/:id                    # Обновить (ADMIN)
DELETE /news/:id                    # Удалить (ADMIN)
```

Query параметры для GET /news:
- `categoryId` — фильтр по категории
- `search` — поиск по заголовку/описанию

### Ресурсы

```
GET    /resources                   # Список ресурсов
GET    /resources/:id               # Ресурс по ID
POST   /resources                   # Создать (ADMIN)
PATCH  /resources/:id               # Обновить (ADMIN)
DELETE /resources/:id               # Удалить (ADMIN)
```

### Категории

```
GET    /categories                  # Список категорий
POST   /categories                  # Создать (ADMIN)
PATCH  /categories/:id              # Обновить (ADMIN)
DELETE /categories/:id              # Удалить (ADMIN)
```

### Уведомления

```
POST /notifications/send            # Отправить (ADMIN)
POST /notifications/register-token  # Зарегистрировать device token
```

## Для диплома

### Актуальность

В современных условиях цифровизации образования существует потребность в удобном мобильном инструменте для оперативного информирования участников учебного процесса. Традиционные методы распространения информации — email-рассылки, объявления на сайте — не обеспечивают необходимой оперативности и доступности. Разработка мобильного приложения для вуза позволяет создать единую точку доступа к актуальным новостям и образовательным ресурсам.

### Цель

Разработать мобильное приложение для студентов и преподавателей высшего учебного заведения, обеспечивающее оперативный доступ к новостям, образовательным ресурсам и системе push-уведомлений.

### Задачи

1. Проанализировать существующие решения в области мобильных образовательных платформ
2. Спроектировать архитектуру системы и схему базы данных
3. Разработать RESTful API на Node.js с использованием Prisma ORM
4. Реализовать кросс-платформенное мобильное приложение на React Native
5. Интегрировать систему push-уведомлений
6. Реализовать ролевую модель доступа
7. Протестировать и задокументировать систему

### Объект исследования

Процесс информационного взаимодействия участников образовательного процесса в высшем учебном заведении.

### Предмет исследования

Технологии разработки кросс-платформенных мобильных приложений для образовательных учреждений с использованием React Native.

### Практическая значимость

Разработанное приложение может быть внедрено в любом высшем учебном заведении для организации оперативного информирования студентов и преподавателей. Модульная архитектура системы позволяет легко расширять функциональность: добавлять новые типы контента, интегрировать с существующими системами управления учебным процессом (LMS), реализовывать дополнительные роли пользователей.
