# Руководство по демонстрации

## Подготовка к демонстрации (за 5 минут до защиты)

1. Запустить PostgreSQL (должен работать)
2. Открыть терминал 1 → `cd backend && npm run dev`
3. Открыть терминал 2 → `cd mobile && npx expo start --android`
4. Дождаться загрузки приложения на эмуляторе
5. Держать в памяти: логин `student@university.ru` / `password123`

---

## Сценарий демонстрации

### Шаг 1: Запуск (30 секунд)

- Показать заставку «УниВест» на синем фоне
- Показать экран входа с логотипом и формой
- Объяснить: _«Это мобильное приложение для студентов и преподавателей. Авторизация по email и паролю с JWT-токенами»_

---

### Шаг 2: Вход как студент (1 минута)

Ввести: `student@university.ru` / `password123`

**Главный экран:**
- Показать приветствие с именем и ролью «Студент»
- Показать блок последних новостей с картинками
- Показать блок образовательных ресурсов
- Объяснить: _«Данные загружаются с backend через REST API»_

---

### Шаг 3: Новости (2 минуты)

Нажать вкладку «Новости»:
- Показать список карточек с категориями и изображениями
- Нажать на категорию «Мероприятия» — список отфильтруется
- Нажать на категорию «Все» — вернётся полный список
- Ввести в поиск «конференция» — найдёт нужные новости
- Очистить поиск
- Открыть любую новость — показать полный текст, автора, дату
- Объяснить: _«Фильтрация и поиск работают через query параметры API»_

---

### Шаг 4: Образовательные ресурсы (1 минута)

Нажать вкладку «Ресурсы»:
- Показать список ресурсов с иконками типов (ссылка/файл/текст)
- Нажать фильтр «Библиотека»
- Открыть ресурс «Электронная библиотека» — показать описание
- Нажать «Открыть ресурс» — откроется браузер
- Объяснить: _«Приложение открывает ссылки через Linking API React Native»_

---

### Шаг 5: Профиль (30 секунд)

Нажать вкладку «Профиль»:
- Показать аватар с инициалами, имя, роль «Студент», факультет
- Объяснить систему ролей: STUDENT, TEACHER, ADMIN

---

### Шаг 6: Выход и вход как администратор (30 секунд)

- Нажать «Выйти из системы» → подтвердить
- Войти: `admin@university.ru` / `password123`

---

### Шаг 7: Административная панель (2 минуты)

Теперь видно другой набор вкладок (Новости, Ресурсы, Категории, Рассылка, Профиль).

**Управление новостями:**
- Показать список всех новостей (включая черновики с жёлтым индикатором)
- Нажать «+» для создания новости
- Заполнить форму прямо на защите: заголовок, описание, текст
- Выбрать категорию
- Включить «Опубликовать»
- Нажать «Создать новость»
- Показать что новость появилась в списке
- Объяснить: _«Все операции создания/изменения/удаления — это запросы к backend API с проверкой роли ADMIN»_

**Управление категориями:**
- Показать список категорий с цветовой кодировкой
- Нажать «+» → заполнить модальное окно
- Сохранить — категория появится в списке

**Отправка уведомлений:**
- Показать экран рассылки
- Нажать один из шаблонов — поля заполнятся
- Показать предпросмотр уведомления
- Объяснить: _«Уведомления отправляются через Expo Push API на все зарегистрированные устройства»_

## Вопросы комиссии и ответы

**— Почему React Native, а не нативная разработка?**
> React Native позволяет писать один код для iOS и Android. При нативной разработке нужно два отдельных приложения на Swift и Kotlin. Для учебного проекта это избыточно, а результат идентичен.

**— Как работает авторизация?**
> После входа backend выдаёт JWT токен — это подписанная строка с ID пользователя и его ролью. Токен сохраняется в зашифрованном хранилище телефона (SecureStore). При каждом запросе к API токен передаётся в заголовке. Backend проверяет подпись и извлекает данные пользователя.

**— Как обеспечивается безопасность?**
> Пароли хранятся только в виде bcrypt-хешей, никогда в открытом виде. JWT токен подписан секретным ключом — подделать его нельзя. Все чувствительные маршруты защищены middleware, проверяющим токен и роль.

**— Можно ли добавить новую функцию?**
> Да, архитектура модульная. Для нового ресурса нужно: добавить модель в Prisma schema, создать миграцию, написать модуль с router/controller/service, добавить экраны в мобильное приложение.

**— Почему PostgreSQL, а не MongoDB?**
> Данные имеют чёткую реляционную структуру: новости → категории → авторы. PostgreSQL с Prisma даёт типобезопасные JOIN-запросы и гарантии целостности данных через foreign keys.

**— Как работают push-уведомления?**
> Когда пользователь входит в приложение, Expo регистрирует устройство и выдаёт уникальный Expo Push Token. Этот токен сохраняется на сервере. Когда администратор нажимает «Отправить», сервер собирает все токены из БД и делает запрос к Expo Push API, который доставляет уведомления на устройства.
