# Текст для дипломной работы

## Актуальность

В условиях стремительной цифровизации образовательной среды возрастает потребность в эффективных инструментах информационного взаимодействия между участниками учебного процесса. Традиционные каналы распространения информации в высших учебных заведениях — информационные стенды, электронная почта, сайты — обладают рядом существенных недостатков: низкой оперативностью доставки, необходимостью активного поиска информации пользователем, отсутствием персонализации.

Согласно статистике, более 95% студентов ежедневно используют смартфон. Мобильное приложение позволяет доставлять актуальную информацию напрямую на устройство пользователя в режиме реального времени с помощью push-уведомлений.

Разработка специализированного мобильного приложения для нужд вуза является актуальной задачей, позволяющей повысить уровень информированности студентов и преподавателей, ускорить распространение важных объявлений и упростить доступ к образовательным ресурсам.

## Цель работы

Целью дипломной работы является проектирование и разработка мобильного приложения для рассылки новостей и образовательных ресурсов высшего учебного заведения с использованием технологии React Native.

## Задачи

1. Проанализировать существующие решения в области мобильных информационных систем для образовательных учреждений
2. Определить требования к функциональности системы на основе потребностей студентов, преподавателей и административного персонала
3. Спроектировать архитектуру системы, включающей мобильное приложение, серверную часть и базу данных
4. Разработать схему базы данных и реализовать серверное API с использованием Node.js, Express и Prisma ORM
5. Реализовать кросс-платформенное мобильное приложение на React Native с полным пользовательским интерфейсом
6. Интегрировать систему push-уведомлений для оперативного информирования пользователей
7. Реализовать ролевую модель разграничения доступа: студент, преподаватель, администратор
8. Наполнить систему тестовыми данными и провести функциональное тестирование
9. Разработать техническую документацию проекта

## Описание системы

Разработанная система состоит из трёх основных компонентов:

### Мобильное приложение

Реализовано на платформе React Native с использованием Expo SDK, что обеспечивает совместимость с операционными системами Android и iOS при единой кодовой базе. Пользовательский интерфейс построен с применением компонентной архитектуры React.

Навигация организована с помощью библиотеки React Navigation и включает стековую навигацию для перехода между экранами и навигацию с нижними вкладками для основных разделов приложения. Управление состоянием реализовано на основе библиотеки Zustand.

### Серверная часть (Backend API)

Сервер реализован на платформе Node.js с использованием фреймворка Express. RESTful API обеспечивает весь необходимый функционал для работы мобильного приложения. Валидация входных данных осуществляется библиотекой Zod.

Авторизация построена на стандарте JSON Web Tokens (JWT). При входе пользователя сервер генерирует подписанный токен, содержащий идентификатор пользователя и его роль. Токен передаётся с каждым запросом и проверяется middleware-функцией.

Разграничение доступа реализовано через roleMiddleware — промежуточный обработчик, проверяющий роль пользователя перед выполнением операций, требующих прав администратора.

### База данных

Для хранения данных используется СУБД PostgreSQL. Схема базы данных включает шесть таблиц: пользователи, новости, ресурсы, категории, уведомления и токены устройств. Работа с базой данных осуществляется через Prisma ORM, обеспечивающий типобезопасный доступ к данным и управление миграциями.

## Описание технологий

### React Native
Фреймворк с открытым исходным кодом, разработанный компанией Meta (Facebook). Позволяет создавать нативные мобильные приложения для iOS и Android с использованием JavaScript/TypeScript. Компоненты React Native транслируются в нативные UI-элементы платформы, что обеспечивает производительность, сопоставимую с нативными приложениями.

### TypeScript
Строго типизированное надмножество JavaScript, разработанное Microsoft. Добавляет статическую типизацию, интерфейсы и другие возможности, позволяющие обнаруживать ошибки на этапе разработки. Применяется как в серверной, так и в клиентской части проекта.

### Node.js + Express
Node.js — среда выполнения JavaScript на стороне сервера, основанная на движке V8. Express — минималистичный веб-фреймворк для Node.js, предоставляющий инструменты для построения REST API: маршрутизацию, middleware, обработку запросов и ответов.

### PostgreSQL + Prisma
PostgreSQL — объектно-реляционная система управления базами данных с открытым исходным кодом. Prisma — современный ORM для Node.js, генерирующий типобезопасный клиент на основе декларативной схемы базы данных.

### JWT (JSON Web Tokens)
Открытый стандарт (RFC 7519) для передачи заявлений между сторонами в виде JSON-объекта. Токен состоит из трёх частей: заголовка, полезной нагрузки и подписи. Используется для реализации аутентификации без состояния (stateless authentication).
